# Security Incident Runbook

## Severity Levels

| Level | Definition                        | Contain By |
|-------|-----------------------------------|------------|
| P0    | Active breach / confirmed data loss | ≤1 hour   |
| P1    | Active exploit / unauthorized access| ≤4 hours  |
| P2    | Confirmed vulnerability, unexploited| ≤24 hours |
| P3    | Suspected vulnerability             | ≤72 hours |

## Roles

- Incident Commander: owns response, declares severity
- Security Lead: technical containment
- Privacy Officer: legal obligations assessment
- Communications Lead: stakeholder + regulatory notifications

## Escalation Chain

Security Lead → Incident Commander → Privacy Officer → Legal Counsel

## Response Timeline per Severity (P0 example)

- T+0h: Incident detected → notify Incident Commander
- T+1h: Containment complete or escalate to CEO
- T+24h: Root cause analysis initiated
- T+72h: Preliminary report to relevant parties if RROSH

## Post-Incident Record Retention

Minimum 24 months.
